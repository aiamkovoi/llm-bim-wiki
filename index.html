LLM Wiki: BIM Knowledge Base

Turn your BIM documents into a living knowledge base. It answers questions by role and remembers project patterns.

Unlike standard AI chatbots or pure vector RAG, this app doesn't just run semantic search on raw files. It compiles your documents into an evolving, human-readable intermediate knowledge base. Read Karpathy's explanation

Based on Karpathy's LLM Wiki pattern · adapted for BIM teams · open source

What It Does
Turn your BIM documents into a structured wiki. It tracks recurring patterns and lets you ask questions by role.
Browser-only MVP
BCF parsingReads native BCF issues and turns them into structured text for wiki ingest.
Discipline batchingSplits larger BCF files into discipline groups to keep context focused.
Multi-projectEach project keeps its own wiki pages, ingest state and saved insights.
Backup / restoreExport the whole workspace and import it later without a backend.

Step 1: Parse documents
Upload BIM documents (BCF, BEP, meeting minutes, standards). The LLM reads them and builds a structured wiki, linking issues and decisions across files. Each new upload adds to what the wiki already knows.
When to run:
🚀 Project start - upload BEP, standards, initial BCF log
📋 After each coordination meeting - add meeting minutes
🔄 When BCF is updated - add new or closed issues
📄 When BEP is revised - update standards and LOD matrix
You don't need to run it every time - only when new documents arrive.
